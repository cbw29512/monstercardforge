# UI QA Audit

## Issue

The current app looked like an engineering prototype. It was technically functional but not yet easy enough to navigate or polished enough for a product that will sell printable cards.

## Art Issue

Cards used placeholder text because the project needs original, licensed, or generated/owned artwork before selling. Until then, the card engine now uses themed placeholder art compositions so cards look more like cards while remaining safe.

## UI Problems Found

- Navigation did not guide the user toward the Free Goblin Sample.
- The sidebar felt like a database tool instead of a product experience.
- The homepage did not immediately explain the product.
- Card previews lacked visual impact without art.
- Filters appeared before the user understood why they mattered.
- The app needs a cleaner buyer path: sample -> browse -> homebrew -> print.

## UI Direction

- Stronger landing section.
- Clear primary CTA: Print Free Goblin Card.
- Better visual hierarchy.
- Larger card previews.
- Cleaner navigation labels.
- Better product copy.
- Guided browsing by ruleset, type, CR, and pack.
- Clear print instructions close to the cards.

## Current Fix

- Added safe themed placeholder art.
- Improved front card visual identity.
- Next step: replace the app shell with a simpler customer-first layout.
