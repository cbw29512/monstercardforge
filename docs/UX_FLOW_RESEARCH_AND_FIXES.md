# UX Flow Research and Fixes

## Research Notes

The landing page must immediately answer three questions:

1. What problem does this solve?
2. Who is it for?
3. What should I do next?

For Monster Card Forge, the core problem is combat friction: Game Masters lose time flipping books, switching tabs, checking monster actions, and tracking boss rules.

## Current Fix

The hero now leads with the problem and outcome:

- Stop flipping books in combat.
- Print the monster card.
- Run the encounter faster.

## Customer Path

The first visit should naturally flow like this:

1. Understand the problem.
2. See the free Goblin card.
3. Print the sample.
4. Understand fold-over vs Boss Folio.
5. Browse Dragon and Lich examples.
6. Create homebrew if desired.
7. Eventually buy/download packs.

## Tab Purpose

### Free Goblin Sample

Landing page and product demo. It should explain the problem, show the solution, and let the user print quickly.

### Browse Cards

Card library. It should answer: what monster do I need tonight?

### Create Homebrew

Guided builder. It should answer: how do I make my own card without formatting knowledge?

### Legal / Attribution

Trust page. It should answer: is this legally clear and properly separated by ruleset/source?

## Next UX Tasks

- Add card-pack browsing cards on the landing page.
- Add clearer visual buttons for Fold-Over vs Boss Folio.
- Add a top-level Print Studio tab when print modes mature.
- Add user-friendly empty states for 2024 ruleset until data is loaded.
- Add SEO page templates for product/pack pages.
